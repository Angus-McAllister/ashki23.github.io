Helpful functions in R

Ashkan Mirzaee

R is a powerful tool for statistical learning, data analysis and graphics and includes numerous built-in functions. R is a very self explanatory environment in terms of documentation. We can learn about any R commands only by using help(command). The following are great resources to learn R programming:

  • Basic graphics in R
  • R Tutorial
  • Programming with R
  • R for Reproducible Scientific Analysis

In this article we will learn about some fundamental functions for data structures, date and time data, data mining, control flow and defining new functions.

Operators

R arithmetic operators are + - * / ^ % any %. For example:

a %/% b  # Intiger division
a %% a   # Remainder
A %*% B  # Matrix multiplication
a ^ n    # nth power 
a ^ 1/n  # nth root

The other operators include:

  • Negation: !
  • Indexing: [ [[
  • Sequence operator: :
  • Component/slot extraction: $ @
  • Logical: & && (and) | || (or)
  • Membership: %in%
  • Assignment: = <- ->
  • Ordering and comparison: < > <= >= == !=

Data structures

There are four major data structure in R:

  • Vectors: c/cbind/rbind
  • Matrices: matrix
  • Data frames: data.frame
  • Lists: list

c combines values into a vector. cbind (rbind) takes a sequence of vector, matrix or data-frame arguments and combine by columns (rows), respectively. This function keeps all array with a same type (e.g all int or char). matrix creates a matrix from the given set of values. In matrix we can use [ , ] to call columns and rows.

data.frame creates data frames, store each column separately as a different variable with different observations (n obs. of m variables). When we read a csv file it saves as a dataframe. In data frames we can use $ to call columns and [ , ] for calling a certain index.

list makes a data structure that each row is not necessarily equal. For example if you have a loop that do not generate same amount of results at each iteration then you can store them in a list format. In lists we can use $ or [[ ]] to call components and [ ] to call members of each component.

Matrix operations

matrix function creates a matrix form of a given set of values. For example matrix(c(1,2,3,3,2,1), 2, 3) generates a \(\small 2 \times 3\) matrix such as:

##      [,1] [,2] [,3]
## [1,]    1    3    2
## [2,]    2    3    1

The most common matrix operations include:

  • Dimensions of a matrix: dim()
  • Diagonal of a matrix: diag()
  • Identity matrix: diag(number)
  • Matrix symmetric test: isSymmetric()
  • Matrix multiplication: %*%
  • Test matrices for exact equality: identical(,)
  • Matrix transpose: t()
  • Determinant of a matrix: det()
  • Inverse of a matrix: solve()
  • Solve a system of equations: solve()
  • QR Decomposition of a Matrix: qr()
  • Spectral Decomposition of a Matrix: eigen()

Split and merge data

split function divides the data into the defined groups. merge function merge two data frames by common columns or row names. merge is similar to JOIN in SQL:

  • Inner join: merge(x, y, by)
  • Left join: merge(x, y, by, all.x = T)
  • Right join: merge(x, y, by, all.y = T)
  • Outer join: merge(x, y, by, all = T)

Regular expressions

Regular expressions (regex) are a simple way to find patterns in text. The most common functions implementing regex in R are in the global regular expression print (grep) family. For more information see help("regex") and help(grep). Learn more about regex functions in R at D-RUG. Following are some simple examples.

sub("a", "#", "abcdfa") # Substitution: substitue first "a" in the expression "abcdfa" with "#" 
## [1] "#bcdfa"
gsub("a", "#", "abcdfa") # General substitution: substitue all "a" in the expression "abcdfa" with "#"  
## [1] "#bcdf#"
gsub("//#", "a", "//#bcdf//#") # For "$","*","+",".","?","[","^","{","|","(", and "\\" need to add "//" before!
## [1] "abcdfa"
substr("abcdef", 1, 3) # substr(x, start, stop)
## [1] "abc"
substr("abcdef", 1, regexpr("d","abcdef")[1]) # From first to "d"
## [1] "abcd"
substr("abcdef", nchar("abcdef")+1-3, nchar("abcdef")) # First 3 caracters from right
## [1] "def"
grep("a", c("abc", "def", "cba a", "aa"), value = FALSE) # Shows arrays' number that include "a"
## [1] 1 3 4
grep("a+", c("abc", "def", "cba a", "aa"), value = TRUE) # Shows expressions that include one or more "a"
## [1] "abc"   "cba a" "aa"
grepl("Ashkan", "my name is Ashkan") # True if "Ashkan" is in the text
## [1] TRUE
grepl("^Ashkan", "my name is Ashkan") # True if "Ashkan" is at the begining of the text
## [1] FALSE
grepl("Ashkan$", "my name is Ashkan") # True if "Ashkan" is at the end of the text
## [1] TRUE
grepl("[A-a]shkan", "my name is ashkan") # True if "Ashkan" or "ashkan" is in the text
## [1] TRUE
pattern <- "(\\d{3})[-. )]*(\\d{3})[-. ]*(\\d{4})"

Control flow tools

These statements allow us to control flow of the R script. The most common control statements are:

  • if, else
  • for
  • while
  • break
  • return
  • repeat

The following are some simple examples of using these statements in R.

n <- 10
if (n == 7) {
  print("n is equal 7")
} else if (n > 7) {
  print("n is greater than 7")
} else {
  print("n is smaller than 7")
}

n <- 7  
while (n < 10) {
  print(n)
  n <- n + 1

Defining functions

In R we can define any function that we need. Defining functions, let us use recursive algorithm which is a powerful algorithm to solve the problems. The following show some examples.

# Norm function
norm <- function(x) {
  sqrt(x%*%x)
}
norm(1:4)
##          [,1]
## [1,] 5.477226
# Recersive function that compute n!
fact <- function(n) { 
  if (n == 1) 
    1
  else 
    n * fact(n-1) # Recursive function
}
fact(8)
## [1] 40320
# Recersive function that compute matrix power
matrix.power <- function(p, n) { 
  if (n == 1) 
    p
  else 
    p %*% matrix.power(p, n-1) # Recursive function
}
matrix.power(matrix(c(4,2,2,4), 2, 2), 3)
##      [,1] [,2]
## [1,]  112  104
## [2,]  104  112
# Matrix symmetric test
sym <- function(a) {
  if (is.matrix(a) == TRUE) {
    if (identical(a, t(a)) == TRUE) {
      print("Matrix is symmetric")
    } else print("Matrix is not symmetric")
  } else print("Entery is not a Matrix")
}
sym(matrix(c(4,2,2,4), 2, 2))
## [1] "Matrix is symmetric"
