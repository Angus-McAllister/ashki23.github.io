A comparison between R and Python

Ashkan Mirzaee

R and Python are two different worlds but at the same time they are similar to each other to some extent. Both are very powerful programming languages with different advantages. R is a great platform for statistical analysis, generating graphs, maps and html files while Python gives more freedom to programmers to utilize their thoughts. In this article we compare basic operations of these two languages.

R

R is a powerful programming language for statistical learning analysis, data analysis and graphics. RStudio, the most popular R IDE, is a great platform for using R that provides more computational and graphical features to us. R Markdown, in RStudio, let us to weave together narrative text and code to produce elegantly formatted output and use multiple languages including R, Python, and SQL.

R and RStudio both are open source and free programs that are available online to download. In order to use RStudio, we need to install both R and RStudio.

Operators

R arithmetic operators are + - * / ^ % any %. For example:

a %/% b  # Intiger division
a %% a   # Remainder
A %*% B  # Matrix multiplication
a ^ n    # nth power 
a ^ 1/n  # nth root

The other operators include:

  • Negation: !
  • Indexing: [ [[
  • Sequence operator: :
  • Component/slot extraction: $ @
  • Logical: & && (and) | || (or)
  • Membership: %in%
  • Assignment: = <- ->
  • Ordering and comparison: < > <= >= == !=

Data structures

There are four major data structures in R:

  • c/cbind/rbind
  • matrix
  • data.frame
  • list

c combines values into a vector. cbind (rbind) takes a sequence of vector, matrix or data-frame arguments and combine by columns (rows), respectively. This function keeps all array with a same type (e.g all int or char). matrix creates a matrix from the given set of values.

data.frame creates data frames, store each column separately as a different variable with different observations (n obs. of m variables). When we read a csv file it saves as a dataframe.

list makes a data structure that each row is not necessarily equal. For example if you have a loop that do not generate same amount of results at each iteration then you can store them in a list format.

  • if, else

n <- 10

  print("n is greater than 7")

  print("n is smaller than 7")

Defining functions

By using function command we can define our own functions in R. Each function has a certain number of variables. For instance, function \(\Delta = b^2 - 4ac\) needs three variables to compute the result. The following show some simple examples.

# Delta
delta <- function(a, b, c) {
  b^2 - 4*a*c
}
delta(a = 2, b = 3, c = 4)
## [1] -23
# Norm
norm <- function(x) sqrt(x %*% x)
norm(1:4)
##          [,1]
## [1,] 5.477226
# Square
square <- function(x) return (x * x)
square(2)
## [1] 4
# Factorial
fact_iter <- function(n) {
  p <- 1
  for (i in 1:n) {
     p <- p * i # Not recursive 
  }  
  return(p)
} 
fact_iter(8)
## [1] 40320

We can define recursive functions in R such that:

# Factorial
fact <- function(n) { 
  if (n == 1) 
    1
  else 
    n*fact(n-1) # Recursive function
}
fact(8)
## [1] 40320
# a * b
mult <- function(a, b) {
  if (b == 1) {
  return(a)
  } else {
    return(a + mult(a, b-1)) # Recursive function
  }
}
mult(6, 5)
## [1] 30

Reading and writing

In R we can use read. and write. to read and write the file types that we want.

gpa <- data.frame(name = c("Ashki", "Ari", "Dori", "Pishi"), gpa = c(3.4,3.7,3.9,3.5))

# write
write.table(gpa, file = "~/Documents/gpa.txt", sep = " ", row.names = FALSE, col.names = TRUE)

# add
write.table(data.frame(name = "Ellie", gpa = 3.3), file = "~/Documents/gpa.txt", append = TRUE, sep = " ", row.names = FALSE, col.names = FALSE)

# read
read.table("~/Documents/gpa.txt", header = T)

# csv
write.csv(gpa, file = "~/Documents/gpa.csv", row.names = FALSE)
read.csv("~/Documents/gpa.csv") # header is TRUE by default

Data structures

There are four major data structures in Python:

  • Lists: list() []
  • Tuples: tuple() ()
  • Sets: set()
  • Dictionaries: dict() {}

Lists are mutable, and their elements are usually homogeneous and are accessed by iterating over the list.

A tuple consists of a number of values separated by commas. Though tuples may seem similar to lists, they are often used in different situations and for different purposes. Tuples are immutable, and usually contain a heterogeneous sequence of elements that are accessed via unpacking or indexing. Tuples may be input with or without surrounding parentheses.

A set is an unordered collection with no duplicate elements. Basic uses include membership testing and eliminating duplicate entries. Set objects also support mathematical operations like union, intersection, difference, and symmetric difference.

Unlike sequences (lists, and tuples), which are indexed by a range of numbers, dictionaries are indexed by keys, which can be any immutable type (e.g. strings and numbers). The main operations on a dictionary are storing a value with some key and extracting the value given the key.

# Lists
a = [-1, 1, 23.23, -7.23]
print(type(a))
## <class 'list'>
print(a[0])
## -1
# Tuples
t = 12345, 54321, 'hello!'
print(type(t))
## <class 'tuple'>
print(t[0])
## 12345
# Sets
s = set('python')
print(s) # Unique letters in s 
## {'t', 'o', 'y', 'p', 'n', 'h'}
s2 = {'python', 'anaconda', 'python'}
print(s2)
## {'anaconda', 'python'}
print(type(s2))
## <class 'set'>
# Dictionaries
grades = {'jack': 60, 'joe': 70}
print(type(grades))
## <class 'dict'>
print(grades['jack'])
## 60

Defining a functions

We can create any function in Python. The following are some simple examples.

def delta(a, b, c): # Delta
  return b**2 - 4*a*c
print(delta(a = 2, b = 3, c = 4))
## -23
def norm(x): # Norm
  return (sum(i**2 for i in range(x+1)))**0.5
print(norm(4))
## 5.477225575051661
def square(x): # Square
  return x * x
print(square(2))
## 4
def fact_iter(n): # Factorial
    p = 1
    for i in range(1, n+1):
        p *= i # Not recursive 
    return p 

print(fact_iter(8))
## 40320

We can, also, define recursive functions in Python such that:

def fact(n): # Factorial
    if n == 1:
        return 1
    else:
        return n*fact(n-1) # Recursive function
print(fact(8))
## 40320
def mult(a, b): # a * b
    if b == 1:
        return a
    else:
        return a + mult(a, b-1) # Recursive function
print(mult(6, 5))
## 30
